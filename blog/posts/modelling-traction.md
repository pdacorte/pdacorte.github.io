# Frameworks and Models

While searching for tools to help me be consistent, even when exhausted, I’ve come across a model that can be used to describe the composing elements of traction.

```
Energy + Clarity + Willpower x Focus = Traction/Output
```

Let’s expand on these elements by breaking them down:

| Elements  | Gained By                                                               | Spent By                                                     |
| :--------: | :----------------------------------------------------------------------: | :-----------------------------------------------------------: |
| **Energy**    | True Rest (Meditation / Napping / Deep Sleep) / Maintaining posture     | Any Action / Task Switching                                  |
| **Clarity**   | Writing down your objectives (precise and granular)                     | Not being diligent / Not being consistent                    |
| **Willpower** | Visualization / Avoiding Excess (Food, Stimuli) / Meditation / Exercise | Using Devices / Sugars / Alcohol / Mistreating body and mind |

<br>

Breaking traction down to three constituents and focusing on the gain/loss of each makes it easier to focus on consistently creating quality output, no matter what the project is. 

Here, focus signals the time spent on anything. The more energy, clarity or willpower you start working with, the higher yield per unit of time spent on a task. If that's important for you, start working on those.

# Working with Markdown

One addendum to this blog is that I'm writing everything in Markdown now, which is why I chose to use **Marked.JS** for rendering these blogs. 

For a couple of years now it's been my tool of choice for quick notetaking, especially since I can use Google Colab to render and share them quickly with colleagues. The added bonus of running code from Jupyter Notebooks is just the cherry on top!

The ease of formatting, building tables and lists, headers and subheaders, just creates such a nice and quick logical flow to a document. 

I've also had the luck that most pushes for agent documentation and standardization are all written in Markdown. It takes 5 minutes to learn, pays dividends for a long time.
